## C# Nullable Types

In C#, **Value Types** (like `int`, `double`, `bool`, `DateTime`) **cannot be null** by default. They always have a default value (e.g., `int` defaults to `0`, `bool` defaults to `false`).

However, sometimes you need to represent **"no value"** (e.g., a user didn't answer a question, or a database field is empty). This is where **Nullable Types** come in.

### 1. Declaration (`?` Syntax)

To make a value type accept `null`, add a question mark `?` after the type name.

C#

```C#
// Standard int: CANNOT be null
int number = 0; 
// number = null; // Compile Error!

// Nullable int: Can be a number OR null
int? optionalNumber = null; 
optionalNumber = 100; // Valid
optionalNumber = null; // Valid
```

### 2. Properties (`.HasValue` & `.Value`)

A nullable type is actually a struct wrapper. It has two helper properties:

- `.HasValue`: Returns `true` if it holds data, `false` if it is `null`.
    
- `.Value`: Returns the actual data. **Warning:** If you access `.Value` when it is `null`, the program crashes.
    

C#

```C#
int? score = null;

if (score.HasValue)
{
    Console.WriteLine($"Score is: {score.Value}");
}
else
{
    Console.WriteLine("No score available.");
}
```

### 3. The Null Coalescing Operator (`??`)

This is a very common shortcut. It says: **"Use the value on the left. If it is null, use the default value on the right."**

It is used to safely convert a `int?` back to a normal `int`.

C#

```C#
int? userData = null;

// "If userData is null, set savedData to 0"
int savedData = userData ?? 0; 

Console.WriteLine(savedData); // Output: 0

// ---

userData = 55;
// Now userData is NOT null, so it uses 55
savedData = userData ?? 0; 
Console.WriteLine(savedData); // Output: 55
```

### 4. Nullable Reference Types (Strings)

In older C#, `string` could always be null. In **modern C# (8.0+)**, you should explicitly mark reference types as nullable if you intend for them to be null. This helps the compiler warn you about potential crashes.

C#

```C#
// Implies this string should NEVER be null
string name = "Tom"; 

// Implies this string MIGHT be null
string? middleName = null; 
```

### Summary

|**Type**|**Can be null?**|**Example**|
|---|---|---|
|`int`|**No**|`int x = 0;`|
|`int?`|**Yes**|`int? x = null;`|
|`bool?`|**Yes**|`bool? x = null;` (True, False, or Null)|
