## C# Any Method

The `.Any()` method is a part of **LINQ** (Language Integrated Query). It is used to check if a collection contains elements. It returns a `bool` (`true` or `false`).

**Note:** To use it, you must include `using System.Linq;` at the top of your file.

### 1. Two Ways to Use It

#### A. Check if the list is **not empty**

If you call `.Any()` without arguments, it checks if the collection has _at least one_ element.

C#

```C#
List<int> numbers = new List<int> { 1, 2, 3 };
List<int> emptyList = new List<int>();

// Efficient check for "Is Not Empty"
bool hasElements = numbers.Any();    // true
bool isEmpty     = emptyList.Any();  // false

// This is preferred over 'numbers.Count > 0'
```

#### B. Check if **at least one** item matches a condition

You can pass a **Lambda/Predicate** to check for a specific value. It stops searching as soon as it finds the first match (making it very fast).

C#

```C#
List<string> team = new List<string> { "Tom", "Ana", "Bob" };

// "Is there anyone named 'Ana'?"
bool hasAna = team.Any(x => x == "Ana"); // true

// "Is there anyone whose name is longer than 10 letters?"
bool hasLongName = team.Any(x => x.Length > 10); // false
```

### 2. `Any` vs `Exists` vs `Count`

|**Method**|**Usage**|**Works on...**|**Performance**|
|---|---|---|---|
|**`.Any(...)`**|`list.Any(x => x > 5)`|**All Collections** (Array, List, Database)|**Fastest** (stops at first match).|
|**`.Exists(...)`**|`list.Exists(x => x > 5)`|**List Only** (`List<T>`)|Fast, but less flexible than Any.|
|**`.Count(...)`**|`list.Count(x => x > 5) > 0`|All Collections|**Slow**. It counts _every_ match even if you only need one.|

### Summary

- Use `Any()` to check if a list is **not empty**.
    
- Use `Any(x => ...)` to check if a **condition is met** by at least one item.
