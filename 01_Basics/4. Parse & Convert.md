## C# String Conversion: int.Parse & Convert.ToInt32

These are the "unsafe" counterparts to `TryParse`. They are used when you are _confident_ the string is a valid number. If the conversion fails, they **throw an exception**, which will crash your program if not handled.

### 1. int.Parse()

This is the most direct, common method to parse a string. It is fast but strict.

- **Throws `FormatException`** if the string is not a valid number (e.g., `"12a"`).
    
- **Throws `ArgumentNullException`** if the string is `null`.
    

C#

```C#
// 1. Successful Parse
string s1 = "123";
int num1 = int.Parse(s1); // num1 is 123

// 2. Failure: Invalid Format
string s2 = "123x";
try
{
    int num2 = int.Parse(s2); // THROWS FormatException
}
catch (FormatException)
{
    Console.WriteLine("s2 is not a valid number.");
}

// 3. Failure: Null
string s3 = null;
try
{
    int num3 = int.Parse(s3); // THROWS ArgumentNullException
}
catch (ArgumentNullException)
{
    Console.WriteLine("s3 is null.");
}
```

### 2. Convert.ToInt32()

This method is part of the `Convert` class, which handles many types (not just strings). It is slightly more flexible than `Parse`.

- **Throws `FormatException`** if the string is not a valid number (e.g., `"456b"`).
    
- **Handles `null`**: This is the key difference. It **returns 0** if the string is `null` and does _not_ throw an exception.
    

C#

```C#
// 1. Successful Parse
string s1 = "456";
int num1 = Convert.ToInt32(s1); // num1 is 456

// 2. Key Difference: Null Handling
string s2 = null;
int num2 = Convert.ToInt32(s2); // *** DOES NOT THROW ***
                                // num2 is 0 (the default int value)

// 3. Failure: Invalid Format
string s3 = "456b";
try
{
    int num3 = Convert.ToInt32(s3); // THROWS FormatException
}
catch (FormatException)
{
    Console.WriteLine("s3 is not a valid number.");
}
```

### Summary: Parse vs. Convert vs. TryParse

|**Method**|**null Input**|**Invalid Input (e.g., "abc")**|**Recommended Use**|
|---|---|---|---|
|**`int.Parse()`**|Throws `ArgumentNullException`|Throws `FormatException`|When you are 100% sure the string is valid and not `null`.|
|**`Convert.ToInt32()`**|**Returns 0**|Throws `FormatException`|When you are sure the string is valid _if_ it's not `null` (and you are okay with `null` becoming `0`).|
|**`int.TryParse()`**|**Returns `false`**|**Returns `false`**|**This is the safest option.** Use it for user input or any data you don't trust.|
