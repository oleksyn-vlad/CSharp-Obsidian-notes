## C# Optional & Named Parameters

These features allow for flexible method calls, reducing the need for method overloading.

### 1. Optional Parameters

You can define default values for parameters in the method signature. If the caller omits the argument, the default value is used.

**Rule:** Optional parameters must be defined **after** all required parameters.

C#

```C#
public class Logger
{
    // 'prefix' is optional. Default is "INFO"
    public void Log(string message, string prefix = "INFO")
    {
        Console.WriteLine($"[{prefix}] {message}");
    }
}

// Usage
Logger logger = new Logger();

// 1. Omit the optional argument (uses "INFO")
logger.Log("System started"); 

// 2. Provide the argument (overrides default)
logger.Log("System crashed", "ERROR");
```

### 2. Named Parameters

When **calling** a method, you can specify the parameter name followed by a colon `:`. This allows you to pass arguments in **any order**.

C#

```C#
public void CreateProfile(string name, int age, string country)
{
    // ...
}

// Usage
// Standard call (order matters)
CreateProfile("Ana", 25, "USA");

// Named arguments (order does not matter)
CreateProfile(age: 25, country: "USA", name: "Ana");
```

### 3. Combining Both (The Real Power)

Named parameters are most useful when skipping specific optional parameters in a long list.

C#

```C#
public void SaveSettings(bool enableWifi = true, bool enableBluetooth = true, int volume = 50)
{
    // ...
}

// Usage:
// We want to change ONLY 'volume', keeping the defaults for Wifi and Bluetooth.
// Without named parameters, this would be impossible without passing the booleans too.
SaveSettings(volume: 100); 

// We can mix positional and named arguments
// (Positional arguments must come first)
SaveSettings(false, volume: 0); // Sets enableWifi=false, enableBluetooth=default(true), volume=0
```
